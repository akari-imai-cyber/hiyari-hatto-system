# 企業管理者向け ユーザー管理ガイド

## 📋 概要

企業管理者が**自社のユーザーを管理**できる機能です。一般ユーザーを企業管理者に昇格させたり、新しいユーザーを追加できます。

---

## 🎯 できること

### 企業管理者（company_admin）ができること

| 機能 | 説明 |
|------|------|
| 👤 **ユーザー一覧表示** | 自社のユーザー全員を表示 |
| ⬆️ **管理者に昇格** | 一般ユーザー → 企業管理者 |
| ⬇️ **一般に降格** | 企業管理者 → 一般ユーザー |
| ✏️ **ユーザー編集** | 氏名とロールを変更 |
| ➕ **ユーザー追加** | 新しいユーザーを作成 |
| 🗑️ **ユーザー削除** | 不要なユーザーを削除 |

### 制限

- ❌ 他社のユーザーは見えません
- ❌ 企業選択はできません（自社のみ自動表示）
- ❌ システム管理者（admin）は編集できません

### 画面の見方

#### 自動表示される内容

ログイン後、以下が自動的に表示されます：

```
┌─────────────────────────────────────────┐
│ 企業を選択                               │
│ ┌─────────────────────────────────────┐ │
│ │ NLP物流株式会社 (nlp-test) [固定]    │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ NLP物流株式会社 (nlp-test) のユーザー    │
│ [➕ 新しいユーザーを追加]                 │
├─────────────────────────────────────────┤
│ メール              氏名    ロール  操作  │
│ user1@...          山田    管理者  [編集] │
│ user2@...          佐藤    一般   [昇格]  │
└─────────────────────────────────────────┘
```

**ポイント**:
- 企業選択は**固定**（変更不可）
- 自社のユーザー**のみ**表示
- ユーザー一覧は**自動的に読み込まれる**

---

### 一般ユーザーを管理者に昇格

#### ケース: 「佐藤さんを企業管理者にしたい」

1. ユーザー一覧で佐藤さんの行を見つける
2. 「**⬆️ 管理者に昇格**」ボタンをクリック
3. 確認ダイアログで「**OK**」
4. ロールが「👨‍💼 企業管理者」に変わる

**完了！** 佐藤さんは次回ログイン時から以下ができるようになります：
- 自社の報告を編集
- 修正依頼に対応
- ユーザー管理

---

### 企業管理者を一般ユーザーに降格

#### ケース: 「山田さんを一般ユーザーに戻したい」

1. ユーザー一覧で山田さんの行を見つける
2. 「**⬇️ 一般に降格**」ボタンをクリック
3. 確認ダイアログで「**OK**」
4. ロールが「👤 一般ユーザー」に変わる

---

### 新しいユーザーを追加

#### ケース: 「新入社員の田中さんを追加したい」

1. 「**➕ 新しいユーザーを追加**」ボタンをクリック
2. モーダルが開く
3. 以下を入力：
   - **メールアドレス**: `tanaka@nlp-test.com`
   - **パスワード**: `Tanaka2026!`（8文字以上）
   - **氏名**: `田中一郎`
   - **ロール**: `一般ユーザー`（または `企業管理者`）
4. 「**➕ 追加**」ボタンをクリック
5. 「ユーザー tanaka@nlp-test.com を作成しました」と表示される

**完了！** 田中さんに以下を伝えてください：
- **ログインURL**: https://stellular-profiterole-2ff0a2.netlify.app/index.html
- **企業コード**: `nlp-test`
- **メール**: `tanaka@nlp-test.com`
- **パスワード**: `Tanaka2026!`

---

### ユーザー情報を編集

#### ケース: 「佐藤さんの氏名を変更したい」

1. ユーザー一覧で佐藤さんの行を見つける
2. 「**✏️ 編集**」ボタンをクリック
3. 編集モーダルが開く
4. 氏名を変更（例: `佐藤 太郎` → `佐藤 次郎`）
5. ロールも必要に応じて変更
6. 「**💾 保存**」ボタンをクリック

---

### ユーザーを削除

#### ケース: 「退職した鈴木さんを削除したい」

1. ユーザー一覧で鈴木さんの行を見つける
2. 「**🗑️ 削除**」ボタンをクリック
3. **1回目の確認**ダイアログで「**OK**」
4. **2回目の確認**（最終確認）で「**OK**」
5. 「suzuki@... を削除しました」と表示される

**警告**:
- ⚠️ 削除は取り消せません
- ⚠️ 削除前に必要なデータをバックアップしてください

## 💡 よくある操作シナリオ

### シナリオ1: 複数の管理者を設定

**状況**: チームリーダー3人を管理者にしたい

1. ユーザー管理画面を開く
2. 1人目のリーダーの「⬆️ 管理者に昇格」をクリック
3. 2人目のリーダーの「⬆️ 管理者に昇格」をクリック
4. 3人目のリーダーの「⬆️ 管理者に昇格」をクリック

**所要時間**: 約1分

---

### シナリオ2: 新入社員5人を一括追加

**方法**: 1人ずつ追加ボタンで登録

1. 「➕ 新しいユーザーを追加」
2. 1人目の情報を入力 → 「➕ 追加」
3. 再度「➕ 新しいユーザーを追加」
4. 2人目の情報を入力 → 「➕ 追加」
5. 繰り返す

**所要時間**: 約3分/人 = 合計15分

---

### シナリオ3: 退職者の整理

**毎月の定期作業**として推奨

1. ユーザー管理画面を開く
2. 退職者を確認
3. 各退職者の「🗑️ 削除」をクリック
4. 確認して削除

## 🚨 トラブルシューティング

### エラー: 「ユーザー一覧の読み込みに失敗しました」

**原因**:
- ネットワークエラー
- Supabase接続エラー

**解決策**:
1. ページを再読み込み（F5）
2. ログアウトして再ログイン
3. ブラウザのキャッシュをクリア

---

### エラー: 「ユーザーの作成に失敗しました」

**原因**:
- メールアドレスが既に存在する
- パスワードが8文字未満

**解決策**:
1. 別のメールアドレスを使用
2. パスワードを8文字以上に設定
3. 英数字＋記号の組み合わせを使用

---

### エラー: 「管理者権限が必要です」

**原因**:
- ログインユーザーが `company_admin` ではない

**解決策**:
1. ログアウト
2. 企業管理者アカウントで再ログイン
3. それでもエラーの場合、システム管理者に連絡

## 🔒 セキュリティ

### パスワードの推奨設定

| 項目 | 推奨 |
|------|------|
| **最低文字数** | 8文字以上 |
| **組み合わせ** | 英大文字・小文字・数字・記号 |
| **例** | `Abc@1234`, `MyPass!2026` |
| **避けるべき** | `12345678`, `password`, `abcdefgh` |

### ユーザー削除時の注意

- 削除前に以下を確認：
  - ✅ 本当に不要なユーザーか
  - ✅ 作成した報告はどうするか
  - ✅ バックアップは取ったか

## 🔗 関連機能

### 報告管理との連携

ユーザー管理で昇格させた管理者は、以下ができるようになります：

1. **報告管理画面にアクセス**
   - ナビの「📋 報告管理」リンクが表示される

2. **自社の報告を編集**
   - ドライバーの報告を修正
   - 修正依頼に対応

3. **ステータス管理**
   - 未確認 → 確認済み → 完了
