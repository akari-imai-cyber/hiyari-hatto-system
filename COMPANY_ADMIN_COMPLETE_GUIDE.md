## 企業管理者の役割

### 企業管理者とは

企業管理者（company_admin）は、**自社のヒヤリハット報告を管理する責任者**です。

### 権限と責任

| 権限 | 説明 |
|------|------|
| **ユーザー管理** | 自社のユーザーの追加・編集・削除・昇格/降格 |
| **報告閲覧** | 自社のすべてのヒヤリハット報告を閲覧 |
| **報告編集** | 報告内容の修正、ステータスの変更 |
| **修正依頼対応** | ドライバーからの修正依頼への対応 |
| **データエクスポート** | 自社のデータをCSV形式でダウンロード |
| **データ分析** | 自社のヒヤリハット傾向分析 |

### システム管理者との違い

| 項目 | 企業管理者 | システム管理者 |
|------|-----------|---------------|
| **データ範囲** | 自社のみ | 全企業 |
| **企業管理** | 不可 | 可能（企業の追加・削除） |
| **ユーザー管理** | 自社のみ | 全企業 |
| **報告管理** | 自社のみ | 全企業 |

## ユーザー管理

### ユーザー管理画面を開く

1. ログイン後、ナビゲーションの **「👥 ユーザー管理」**をクリック
2. または直接アクセス：
   ```
   https://stellular-profiterole-2ff0a2.netlify.app/admin-users.html
   ```

### ユーザー一覧の確認

自社のユーザーが一覧表示されます：

| 列名 | 説明 |
|------|------|
| **メールアドレス** | ユーザーのログインID |
| **氏名** | ユーザーの名前（未設定の場合は「-」） |
| **ロール** | 👤 一般ユーザー / 👨‍💼 企業管理者 |
| **作成日** | アカウント作成日時 |
| **操作** | 編集・昇格/降格・削除ボタン |

### 新しいユーザーを追加

#### 方法1: Supabase経由（推奨）

**ステップ1: Supabaseでユーザー作成**

1. https://supabase.com にログイン
2. プロジェクトを選択
3. **Authentication → Users → Add user** をクリック
4. 以下を入力：
   - Email: `new-driver@nlp-test.com`
   - Password: `StrongPass123!`
   - ✅ **Auto Confirm User** をチェック
5. **Create user** をクリック
6. 作成されたユーザーの **UID（UUID）**をコピー

**ステップ2: プロフィール情報を追加**

Supabase の **SQL Editor** で以下を実行：

```sql
INSERT INTO public.profiles (id, email, role, company_id, created_at)
VALUES (
    '<コピーしたUID>',
    'new-driver@nlp-test.com',
    'company_user',
    '<あなたの会社のID>',
    NOW()
);
```

**会社IDの確認方法**：

```sql
SELECT id, company_code, company_name 
FROM public.companies 
WHERE company_code = 'nlp-test';
```

#### 方法2: ユーザー管理画面から（開発中）

> 注意: 現在のバージョンでは、Supabaseのメール確認設定により、
> ユーザー管理画面からの直接追加でエラーが発生する場合があります。
> **方法1（Supabase経由）**を推奨します。

### 一般ユーザーを管理者に昇格

1. ユーザー一覧で昇格したいユーザーを見つける
2. **「⬆️ 管理者に昇格」**ボタンをクリック
3. 確認ダイアログで **「OK」**をクリック
4. ロールが **「👨‍💼 企業管理者」**に変更されます

### 管理者を一般ユーザーに降格

1. ユーザー一覧で降格したい管理者を見つける
2. **「⬇️ 一般ユーザーに降格」**ボタンをクリック
3. 確認ダイアログで **「OK」**をクリック
4. ロールが **「👤 一般ユーザー」**に変更されます

### ユーザー情報の編集

1. ユーザー一覧で編集したいユーザーの **「✏️ 編集」**ボタンをクリック
2. 編集モーダルで以下を変更可能：
   - 氏名
   - ロール（一般ユーザー / 企業管理者）
3. **「💾 保存」**ボタンをクリック

### ユーザーの削除

1. ユーザー一覧で削除したいユーザーの **「🗑️ 削除」**ボタンをクリック
2. 1回目の確認ダイアログで **「OK」**をクリック
3. 2回目の確認ダイアログで **「OK」**をクリック
4. ユーザーが削除されます

⚠️ **注意**: 削除されたユーザーはログインできなくなります

## 報告管理

### 報告管理画面を開く

1. ログイン後、ナビゲーションの **「📋 報告管理」**をクリック
2. または直接アクセス：
   ```
   https://stellular-profiterole-2ff0a2.netlify.app/admin-reports.html
   ```

### 報告一覧の見方

自社のすべての報告が表示されます：

| 列名 | 説明 |
|------|------|
| **報告者** | 報告したドライバー名 |
| **発生日時** | ヒヤリハットが発生した日時 |
| **場所** | 発生場所 |
| **カテゴリ** | ヒヤリハットの種類 |
| **ステータス** | 未確認 / 確認済み / 対応中 / 完了 |
| **修正依頼** | 🔴（赤バッジ）= 修正依頼あり |
| **操作** | ✏️ 編集 / 🗑️ 削除 |

### フィルタリング機能

#### ステータスでフィルタ

ドロップダウンから選択：
- すべてのステータス
- 未確認
- 確認済み
- 対応中
- 完了

#### カテゴリでフィルタ

ドロップダウンから選択：
- すべてのカテゴリ
- 運転操作ミス
- 車両不具合
- 交通環境
- その他

### 報告の編集

1. 編集したい報告の **「✏️ 編集」**ボタンをクリック
2. 編集モーダルで以下を変更可能：
   - 報告者名
   - 発生日
   - 発生時刻
   - 発生場所
   - カテゴリ
   - 詳細
   - ステータス
   - 管理者コメント
3. **「💾 保存」**ボタンをクリック

### ステータスの管理

報告のステータスを適切に管理することで、対応状況を可視化できます：

| ステータス | 使用タイミング | 説明 |
|-----------|--------------|------|
| **未確認** | 報告直後 | まだ確認していない報告 |
| **確認済み** | 内容確認後 | 内容を確認し、対応を検討中 |
| **対応中** | 対応開始後 | 改善策を実施中 |
| **完了** | 対応完了後 | すべての対応が完了 |

### 管理者コメントの追加

報告に対する管理者のコメントを追加できます：

1. 報告を編集
2. **「管理者コメント」**欄に記入
   ```
   例：「後続車との車間距離確保の重要性を朝礼で共有済み」
   ```
3. 保存

## 修正依頼への対応

### 修正依頼とは

ドライバーが報告内容の誤りに気づいた場合、修正依頼を送信できます。

### 修正依頼の確認方法

修正依頼がある報告は、報告管理画面で **🔴 修正依頼あり** バッジが表示されます。

### 修正依頼への対応手順

1. **報告管理画面**を開く
2. 🔴 バッジがある報告を見つける
3. **「✏️ 編集」**ボタンをクリック
4. 編集モーダルの下部に **「修正依頼」**セクションが表示されます：
   - 依頼日時
   - 依頼内容（ドライバーのメッセージ）
5. 依頼内容を確認し、報告を修正
6. **「💾 保存」**をクリック
7. 🔴 バッジが自動的に消えます

### 修正依頼の例

**ドライバーの依頼**:
```
発生時刻を14:30から15:00に修正してください。
実際の発生時刻は15:00でした。
```

**対応方法**:
1. 「発生時刻」フィールドを `15:00` に変更
2. 「管理者コメント」に記入：
   ```
   時刻を15:00に修正しました。ご報告ありがとうございます。
   ```
3. 保存

## ベストプラクティス

### 1️⃣ 迅速な対応

- **24時間以内**に新しい報告を確認
- **48時間以内**に修正依頼に対応
- 定期的に報告管理画面をチェック

### 2️⃣ 適切なステータス管理

| ステータス遷移 | タイミング |
|--------------|-----------|
| 未確認 → 確認済み | 報告内容を確認後 |
| 確認済み → 対応中 | 改善策の実施開始 |
| 対応中 → 完了 | すべての対応完了 |

### 3️⃣ 有意義な管理者コメント

良いコメント例：
```
後続車との車間距離確保の重要性について、
2026年3月5日の朝礼で全ドライバーに共有しました。
また、同様の事例を防ぐため、バックミラーの
確認頻度を増やすよう指導しました。
```

悪いコメント例：
```
確認しました。
```

### 4️⃣ 定期的なデータ分析

- **毎週**: 報告件数とカテゴリの確認
- **毎月**: トレンド分析とCSVエクスポート
- **四半期**: 改善策の効果測定

### 5️⃣ ドライバーへのフィードバック

- 報告に対して必ず管理者コメントを記入
- 感謝の言葉を添える
- 改善策を共有

### 6️⃣ ユーザー管理のセキュリティ

- 退職者のアカウントは速やかに削除
- 強力なパスワードを設定
- 管理者権限は必要最小限に

## トラブルシューティング

### ログインできない

**症状**: ログイン画面でエラーが表示される

**対処法**:
1. 企業コードが正しいか確認
2. メールアドレスにスペースや誤字がないか確認
3. ブラウザのキャッシュをクリア（`Ctrl + Shift + Del`）
4. シークレットモードで試す（`Ctrl + Shift + N`）
5. システム管理者に連絡

### ユーザー管理画面が表示されない

**症状**: ナビゲーションに「ユーザー管理」リンクが表示されない

**対処法**:
1. ブラウザでハードリフレッシュ（`Ctrl + Shift + R`）
2. ロールが `company_admin` になっているか確認：
   ```javascript
   // ブラウザのコンソール（F12）で実行
   console.log('Role:', window.currentAuth?.role);
   ```
   期待値: `company_admin`
3. Supabaseでロールを確認・修正

### 他社のデータが見える

**症状**: 自社以外の企業の報告やユーザーが表示される

**対処法**:
1. 最新版がデプロイされているか確認
2. ブラウザキャッシュをクリア
3. シークレットモードで確認
4. システム管理者に連絡

### 修正依頼バッジが消えない

**症状**: 修正して保存したのに🔴バッジが残る

**対処法**:
1. ページをリロード（`F5`）
2. 報告を再度編集して保存
3. ブラウザキャッシュをクリア
4. データベースの `edit_request` フラグを確認

### CSVエクスポートができない

**症状**: エクスポートボタンをクリックしてもファイルがダウンロードされない

**対処法**:
1. ブラウザのポップアップブロックを無効化
2. ダウンロード許可を確認
3. 別のブラウザで試す
4. データ件数が多すぎる場合はフィルタを適用

## よくある質問（FAQ）

### Q1: 企業管理者は何人まで設定できますか？

**A**: 制限はありません。必要に応じて複数の管理者を設定できます。

### Q2: 一般ユーザーも報告を編集できますか？

**A**: いいえ。一般ユーザーは報告の作成と修正依頼のみ可能です。編集は企業管理者の権限です。

### Q3: 削除した報告は復元できますか？

**A**: いいえ。削除した報告は完全に削除されます（ソフトデリート）。重要な報告は削除しないよう注意してください。

### Q4: 他社の報告を見ることはできますか？

**A**: いいえ。企業管理者は自社の報告のみ閲覧可能です。システム管理者のみ全企業の報告を閲覧できます。

### Q5: パスワードをリセットするにはどうすればいいですか？

**A**: 
- **自分のパスワード**: システム管理者に連絡
- **ドライバーのパスワード**: Supabase経由でリセット

### Q6: 報告の入力を代行できますか？

**A**: はい。企業管理者も「報告入力」画面から新しい報告を作成できます。

### Q7: スマートフォンで管理できますか？

**A**: はい。すべての機能がスマートフォンで利用可能です。

### Q8: データのバックアップは必要ですか？

**A**: 定期的にCSVエクスポートを実行し、ローカルにバックアップすることを推奨します。
