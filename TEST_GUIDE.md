## ステップ1: Supabaseデータベースのセットアップ

### 1-1. Supabaseにログイン
1. https://supabase.com にアクセス
2. プロジェクト「NLP状元DB」を開く

### 1-2. 既存データのバックアップ（念のため）
1. 左メニュー → **Table Editor**
2. 「事件」テーブルをクリック
3. 必要なデータがあれば、右上の「...」→「Export to CSV」でバックアップ

### 1-3. 新しいスキーマの実行
1. 左メニュー → **SQL Editor**
2. 「New query」をクリック
3. `SETUP_SUPABASE.sql` の内容を全てコピー＆ペースト
4. **Run** ボタンをクリック
5. 成功メッセージを確認：「テーブル作成完了！incidents テーブルにデータを入力できます。」

### 1-4. テーブルの確認
1. 左メニュー → **Table Editor**
2. 以下のテーブルが表示されることを確認：
   - ✅ **incidents**（新しく作成されたテーブル）
   - ✅ employees
   - ✅ offices
   - ✅ vehicles
3. **incidents** テーブルをクリックして、以下のカラムがあることを確認：
   - id
   - created_at
   - updated_at
   - report_type
   - reporter_name
   - occurred_at
   - location_text
   - incident_type
   - categories（ARRAY型）
   - memo
   - photo_url
   - status
   - その他多数のカラム

## ステップ3: 動作テスト

### テスト1: 新規報告（STEP1のみ）

#### 手順：
1. https://stellular-profiterole-2ff0a2.netlify.app/index.html を開く
2. **Ctrl+Shift+R** で強制リロード
3. **F12** でコンソールを開く（エラーチェック用）
4. 以下を入力：
   - 「🟡 ヒヤリハット」をクリック
   - **報告者名**: テスト太郎
   - **発生日時**: 今日の日付と時刻（例: 2026-02-25 10:30）
   - **発生場所**: テスト営業所前
   - **どちらで起きた？**: 「🚛 走行中」を選択
   - **何が起きた？**: 「急ブレーキ・急ハンドル」を1つ選択（青くハイライト）
   - **補足メモ**: （任意）前方車両が急停止
5. 「**提出する（後から詳細追加可能）**」をクリック

#### 期待される結果：
- ✅ 成功メッセージが表示される
- ✅ 「報告が完了しました」と表示
- ✅ コンソールに赤いエラーが出ない
- ✅ 「保存成功:」というログが表示される

#### もしエラーが出た場合：
- コンソールの赤いエラーメッセージをコピー
- 「Could not find the column 'xxx'」→ Supabaseのテーブル構造が古い可能性
- 「supabaseClient is not defined」→ config.jsの初期化に失敗

## トラブルシューティング

### 問題1: 「Could not find the column 'xxx' of 'incidents'」

**原因**: Supabaseのテーブル構造が古い

**解決方法**:
1. SETUP_SUPABASE.sql を再実行
2. Netlifyを再デプロイ
3. ブラウザキャッシュをクリア（Ctrl+Shift+R）

---

### 問題2: 「supabaseClient is not defined」

**原因**: config.jsの初期化に失敗

**解決方法**:
1. F12 → Console で「アプリケーション初期化完了」が表示されるか確認
2. SUPABASE_CONFIG の URL と anonKey が正しいか確認

---

### 問題3: ダッシュボードに「0件」と表示される

**原因**: データ読み込みに失敗

**解決方法**:
1. F12 → Console で「=== 報告読み込み開始 ===」を確認
2. 「取得データ: Array(0)」→ Supabaseにデータが入っていない
3. 「取得データ: Array(1)」→ データはあるが表示ロジックに問題
4. 赤いエラーがあれば、エラーメッセージを確認

---

### 問題4: 日付が翌日になっている

**原因**: タイムゾーンの変換が正しく動作していない

**解決方法**:
1. main.jsが最新版かチェック
2. Netlifyに最新版をアップロード
3. ブラウザキャッシュをクリア（Ctrl+Shift+R）
