# 報告編集機能 使い方ガイド

**実装日**: 2026年3月1日  
**対象**: 管理者・ドライバー

---

## 📌 概要

ドライバーが入力した報告に誤りがあった場合、以下の2つの方法で対応できます：

1. **ドライバー**: 修正依頼を送信
2. **管理者**: 報告を直接編集

---

## 🚛 ドライバー向け: 修正依頼の送り方

### 手順

1. **ダッシュボードにアクセス**
   ```
   https://stellular-profiterole-2ff0a2.netlify.app/dashboard.html
   ```

2. **修正したい報告を選択**
   - 報告一覧から該当の報告をクリック
   - 詳細モーダルが開きます

3. **修正依頼セクションを見つける**
   - モーダルの下部にスクロール
   - 「修正依頼」セクションがあります

4. **修正依頼内容を入力**
   - テキストボックスに修正してほしい内容を具体的に記入
   - 例: 「発生日時が間違っています。正しくは2月15日14時です。」
   - 例: 「報告者名が誤っていました。正しくは『田中太郎』です。」

5. **「📝 修正依頼を送信」ボタンをクリック**
   - 確認ダイアログで「OK」をクリック
   - 「✅ 修正依頼を送信しました」と表示されます

6. **管理者が対応**
   - 管理者が確認して修正します
   - 修正が完了すると、修正依頼フラグがクリアされます

---

### 修正依頼済みの報告の確認

修正依頼を送信した報告には、詳細モーダルに以下のメッセージが表示されます：

```
⚠️ この報告には修正依頼が送信されています。管理者が対応します。
```

## 👨‍💼 管理者向け: 報告の編集方法

### Step 1: 報告管理画面にアクセス

1. **管理画面にログイン**
   ```
   https://stellular-profiterole-2ff0a2.netlify.app/admin-reports.html
   ```
   - メール: `akari-imai@tehara.co.jp`
   - パスワード: `tehara0425`

2. **報告一覧が表示されます**
   - 全企業の報告が表示されます
   - 修正依頼がある報告には **「修正依頼あり」** バッジが表示されます

---

### Step 2: フィルターで絞り込み（任意）

**企業別フィルター:**
- ドロップダウンから企業を選択
- 特定の企業の報告のみ表示

**ステータス別フィルター:**
- 「基本情報のみ」「詳細入力済」「確認済」など
- 未確認の報告のみ表示したい場合に便利

---

### Step 3: 報告を編集

1. **「✏️ 編集」ボタンをクリック**
   - 編集モーダルが開きます

2. **編集可能な項目:**

   **基本情報:**
   - 報告者名
   - 発生日時
   - 発生場所
   - 報告種別（ヒヤリハット / 事故）
   - インシデント種別（運転中 / 荷役作業中）

   **詳細情報:**
   - メモ・詳細説明
   - 天候
   - 路面状況
   - 重大度（1-5）
   - ステータス

   **管理者メモ:**
   - 管理者コメント
   - 修正依頼フラグをクリア（チェックボックス）

3. **「💾 保存」ボタンをクリック**
   - 「✅ 報告を保存しました」と表示されます
   - 報告一覧に戻ります

---

### Step 4: 修正依頼をクリア

編集が完了したら、修正依頼フラグをクリアします：

1. 編集モーダルの最下部にある **「修正依頼フラグをクリア」** にチェック
2. 「💾 保存」ボタンをクリック
3. 修正依頼バッジが消えます

## 💡 よくある質問

### Q1. ドライバーは自分で報告を編集できますか？

**A**: いいえ、ドライバーは編集できません。修正依頼を送信して、管理者に対応してもらいます。

---

### Q2. 修正依頼を送信したら、報告は削除されますか？

**A**: いいえ、削除されません。修正依頼フラグが立つだけで、報告内容はそのまま残ります。

---

### Q3. 修正依頼を取り消すことはできますか？

**A**: ドライバーからは取り消せません。管理者が編集モーダルで「修正依頼フラグをクリア」のチェックを外して保存すれば取り消されます。

---

### Q4. 複数の報告に修正依頼を送信できますか？

**A**: はい、できます。各報告ごとに修正依頼を送信してください。

---

### Q5. 管理者が編集した内容は、ドライバーに通知されますか？

**A**: 現在、自動通知機能はありません。管理者が「管理者コメント」に編集内容を記入することをおすすめします。

---

### Q6. 編集履歴は記録されますか？

**A**: 現在、編集履歴は記録されていません。`updated_at` フィールドに最終更新日時のみ記録されます。

## 🗄️ データベース構造

### incidents テーブルの新しいカラム

| カラム名 | 型 | 説明 |
|---------|-----|------|
| `edit_request` | BOOLEAN | 修正依頼フラグ（TRUE = 修正依頼あり） |
| `edit_request_message` | TEXT | 修正依頼の内容 |
| `edit_request_date` | TIMESTAMP | 修正依頼の送信日時 |

---

## 🚀 Supabase テーブル拡張手順

### SQL実行

1. Supabase ダッシュボード → SQL Editor
2. `supabase-edit-request.sql` の内容をコピー＆貼り付け
3. 「Run」ボタンをクリック

### 確認クエリ

```sql
-- 修正依頼がある報告を確認
SELECT 
    id,
    reporter_name,
    occurred_at,
    edit_request,
    edit_request_message,
    edit_request_date
FROM public.incidents
WHERE edit_request = TRUE
ORDER BY edit_request_date DESC;
```

## 📝 使用例

### 例1: 発生日時の修正

**ドライバーの修正依頼:**
```
発生日時が間違っています。
正しくは2月15日14時30分です。
```

**管理者の対応:**
1. `admin-reports.html` にアクセス
2. 該当報告の「✏️ 編集」をクリック
3. 「発生日時」フィールドを修正
4. 「管理者コメント」に「発生日時を2/15 14:30に修正しました」と記入
5. 「修正依頼フラグをクリア」にチェック
6. 「💾 保存」をクリック

---

### 例2: 報告者名の修正

**ドライバーの修正依頼:**
```
報告者名が誤っていました。
正しくは「山田太郎」です。
```

**管理者の対応:**
1. `admin-reports.html` にアクセス
2. 該当報告の「✏️ 編集」をクリック
3. 「報告者名」フィールドを「山田太郎」に修正
4. 「管理者コメント」に「報告者名を修正しました」と記入
5. 「修正依頼フラグをクリア」にチェック
6. 「💾 保存」をクリック
