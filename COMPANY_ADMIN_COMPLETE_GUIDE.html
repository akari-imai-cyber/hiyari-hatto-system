企業管理者の役割

企業管理者（company_admin）とは

自社のヒヤリハット報告を管理する責任者です。

権限と責任

👥 ユーザー管理

自社のユーザーの追加・編集・削除・昇格/降格

📋 報告閲覧

自社のすべてのヒヤリハット報告を閲覧

✏️ 報告編集

報告内容の修正、ステータスの変更

📩 修正依頼対応

ドライバーからの修正依頼への対応

📊 データエクスポート

自社のデータをCSV形式でダウンロード

📈 データ分析

自社のヒヤリハット傾向分析

システム管理者との違い

項目 企業管理者 システム管理者
データ範囲 自社のみ 全企業
企業管理 不可 可能（企業の追加・削除）
ユーザー管理 自社のみ 全企業
報告管理 自社のみ 全企業

新しいユーザーを追加（推奨方法）

ステップ1: Supabaseでユーザー作成

  1. https://supabase.com にログイン
  2. プロジェクトを選択
  3. Authentication → Users → Add user をクリック
  4. 以下を入力：
    • Email: new-driver@nlp-test.com
    • Password: StrongPass123!
    • ✅ Auto Confirm User をチェック
  5. Create user をクリック
  6. 作成されたユーザーの UID（UUID）をコピー

ステップ2: プロフィール情報を追加

Supabase の SQL Editor で以下を実行：

INSERT INTO public.profiles (id, email, role, company_id, created_at) VALUES ( '<コピーしたUID>', 'new-driver@nlp-test.com', 'company_user', '<あなたの会社のID>', NOW() );

会社IDの確認方法：

SELECT id, company_code, company_name FROM public.companies WHERE company_code = 'nlp-test';

一般ユーザーを管理者に昇格

  1. ユーザー一覧で昇格したいユーザーを見つける
  2. 「⬆️ 管理者に昇格」ボタンをクリック
  3. 確認ダイアログで 「OK」をクリック
  4. ロールが 「👨‍💼 企業管理者」に変更されます

管理者を一般ユーザーに降格

  1. ユーザー一覧で降格したい管理者を見つける
  2. 「⬇️ 一般ユーザーに降格」ボタンをクリック
  3. 確認ダイアログで 「OK」をクリック
  4. ロールが 「👤 一般ユーザー」に変更されます

ステータスの管理

報告のステータスを適切に管理することで、対応状況を可視化できます：

ステータス 使用タイミング 説明
未確認 報告直後 まだ確認していない報告
確認済み 内容確認後 内容を確認し、対応を検討中
対応中 対応開始後 改善策を実施中
完了 対応完了後 すべての対応が完了

修正依頼への対応

修正依頼とは

ドライバーが報告内容の誤りに気づいた場合、修正依頼を送信できます。

修正依頼がある報告は、報告管理画面で 🔴 修正依頼あり バッジが表示されます。

修正依頼への対応手順

  1. 報告管理画面を開く
  2. 🔴 バッジがある報告を見つける
  3. 「✏️ 編集」ボタンをクリック
  4. 編集モーダルの下部に 「修正依頼」セクションが表示されます
  5. 依頼内容を確認し、報告を修正
  6. 「💾 保存」をクリック
  7. 🔴 バッジが自動的に消えます

修正依頼の例

ドライバーの依頼:

発生時刻を14:30から15:00に修正してください。 実際の発生時刻は15:00でした。

対応方法:

  1. 「発生時刻」フィールドを 15:00 に変更
  2. 「管理者コメント」に記入：
    時刻を15:00に修正しました。ご報告ありがとうございます。
  3. 保存

トラブルシューティング

ログインできない

対処法:

  1. 企業コードが正しいか確認
  2. メールアドレスにスペースや誤字がないか確認
  3. ブラウザのキャッシュをクリア（Ctrl + Shift + Del）
  4. シークレットモードで試す（Ctrl + Shift + N）
  5. システム管理者に連絡

ユーザー管理画面が表示されない

対処法:

  1. ブラウザでハードリフレッシュ（Ctrl + Shift + R）
  2. ロールが company_admin になっているか確認：
    // ブラウザのコンソール（F12）で実行 console.log('Role:', window.currentAuth?.role); // 期待値: company_admin
  3. Supabaseでロールを確認・修正

よくある質問（FAQ）

Q1: 企業管理者は何人まで設定できますか？

A: 制限はありません。必要に応じて複数の管理者を設定できます。

Q2: 一般ユーザーも報告を編集できますか？

A: いいえ。一般ユーザーは報告の作成と修正依頼のみ可能です。編集は企業管理者の権限です。

Q3: 削除した報告は復元できますか？

A: いいえ。削除した報告は完全に削除されます。重要な報告は削除しないよう注意してください。

Q4: 他社の報告を見ることはできますか？

A: いいえ。企業管理者は自社の報告のみ閲覧可能です。システム管理者のみ全企業の報告を閲覧できます。

Q5: スマートフォンで管理できますか？

A: はい。すべての機能がスマートフォンで利用可能です。
