# Supabase メール確認設定の修正

## 🔍 問題

ユーザー追加時に「User already registered」エラーが発生する。

**原因**: Supabase のデフォルト設定では、`signUp` でユーザーを作成すると**メール確認が必要**になり、確認されるまでユーザーが「未確認」状態になります。この状態では同じメールアドレスで再度 `signUp` を呼ぶとエラーになります。

---

## 🔧 解決策：Supabase でメール確認を無効化

### ステップ1: Supabase Dashboard にアクセス

1. https://supabase.com にログイン
2. プロジェクト「NLP運行状況DB」を選択

### ステップ2: Authentication 設定を開く

1. 左サイドバーの **「Authentication」** をクリック
2. 上部タブの **「Providers」** をクリック
3. **「Email」** をクリック

### ステップ3: メール確認を無効化

以下の設定を変更：

| 設定項目 | 変更前 | 変更後 |
|---------|--------|--------|
| **Confirm email** | ✅ Enabled | ❌ Disabled |

**または**:

| 設定項目 | 変更前 | 変更後 |
|---------|--------|--------|
| **Enable email confirmations** | ON | **OFF** |

### ステップ4: 保存

1. 一番下の **「Save」** ボタンをクリック
2. 設定が反映されるまで数秒待つ

---

## ✅ 設定後の動作

### Before（メール確認が有効）
```
1. signUp でユーザー作成
2. ユーザーは「未確認」状態
3. メールに確認リンクが送信される
4. ユーザーがリンクをクリックするまでログインできない
5. 同じメールで再度 signUp → エラー
```

### After（メール確認が無効）
```
1. signUp でユーザー作成
2. ユーザーはすぐに「確認済み」状態
3. メールは送信されない
4. すぐにログイン可能
5. 管理者が作成したユーザーはすぐに使える
```

---

## 🎯 代替案：既存の未確認ユーザーを削除

もしメール確認を無効化したくない場合は、既存の未確認ユーザーを削除してから再作成します。

### Supabase で未確認ユーザーを削除

1. **Authentication** → **Users** タブ
2. 未確認ユーザー（Email confirmed が空欄）を探す
3. ユーザー行の右側 **「...」** メニュー → **「Delete user」**
4. 確認して削除

### SQL で一括削除

```sql
-- 未確認ユーザーを一括削除（注意: 慎重に実行）
DELETE FROM auth.users
WHERE email_confirmed_at IS NULL;
```

## 🔒 セキュリティ上の注意

### メール確認を無効化する影響

| 項目 | メール確認あり | メール確認なし |
|------|--------------|--------------|
| **セキュリティ** | 高い | 中程度 |
| **ユーザー体験** | メール確認が必要 | すぐ使える |
| **推奨シーン** | 公開サービス | 社内システム |

**このシステムの場合**:
- ✅ 社内の物流企業向けシステム
- ✅ 管理者が全ユーザーを作成
- ✅ ユーザーが自分で登録することはない
- ✅ **メール確認を無効化しても問題なし**

## 📝 設定完了後のテスト

1. Supabase でメール確認を無効化
2. ユーザー管理画面で新規ユーザーを追加：
   - メール: `driver1@nlp-test.com`
   - パスワード: `Driver123!`
   - 氏名: `ドライバー1号`
   - ロール: `一般ユーザー`
3. 「ユーザー driver1@nlp-test.com を作成しました」と表示される
4. すぐにそのアカウントでログイン可能

## 🚨 トラブルシューティング

### エラー: 「User already registered」が続く

**原因**: 過去に同じメールアドレスで作成されたユーザーが残っている

**解決策**:
1. Supabase → Authentication → Users
2. 該当メールアドレスのユーザーを削除
3. 再度作成

---

### エラー: 「Invalid email」

**原因**: メールアドレスの形式が正しくない

**解決策**:
- 正しい形式で入力（例: `user@example.com`）
- `@company.local` のようなローカルドメインも使用可能
